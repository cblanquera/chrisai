# Brand Messaging

Use this workflow when the main problem is turning brand strategy and voice
into reusable message assets such as one-liners, value propositions, message
pillars, taglines, CTAs, boilerplate, or website messaging hierarchy.

This workflow produces messaging guidance, not full page copy, ad campaigns,
content strategy, logo concepts, or visual identity.

## Ownership

This workflow owns:

- one-liners and elevator pitches
- value propositions and positioning statements
- message pillars and proof mapping
- tagline and CTA options
- boilerplate descriptions
- message hierarchy for common brand surfaces

This workflow does not own:

- research; use `workflows/brand-research.md`
- strategy foundation; use `workflows/brand-strategy.md`
- voice rules; use `workflows/brand-voice.md`
- long-form copywriting, SEO pages, ads, emails, or sales collateral

## Workflow

Work through these steps in order:

1. Check existing context.
2. Complete only the missing intake.
3. Confirm the primary message job.
4. Build the messaging foundation.
5. Create message assets.
6. Review for clarity, proof, and consistency.
7. State handoff recommendations.

## Step 1: Check Existing Context

Look for existing strategy, voice, customer language, research, homepage copy,
sales decks, product docs, launch notes, or prior messaging docs. Do not assume
a particular folder exists.

Use `workflows/brand-strategy.md` first if positioning, audience, or
differentiation is unclear.

## Step 2: Intake

Capture or infer:

- primary audience
- main conversion or comprehension goal
- category and alternative options
- strongest differentiators
- proof points and credibility signals
- objections or anxieties to address
- preferred tone from the brand voice guide
- required surfaces: website, product listing, deck, social bio, app store,
  docs, sales, investor, or internal

When asking, offer suggestions and ask for available proof before writing bold
claims.

## Step 3: Confirm The Message Job

Define what the messaging must do:

- explain a new category
- make a known category feel differentiated
- build trust
- simplify a complex product
- shift perception during a rebrand
- support launch or fundraising
- guide website or product copy

Do not optimize for cleverness before clarity.

## Step 4: Build The Foundation

Create or refine:

- positioning statement
- core value proposition
- primary customer problem
- desired outcome
- differentiated mechanism or approach
- proof map: claim to evidence
- objection map: concern to response
- audience language to use and avoid

Label unsupported proof or assumptions.

## Step 5: Create Message Assets

Produce only the assets the user needs, or use this default set:

- one-line description
- short elevator pitch
- long elevator pitch
- headline directions
- 3-5 message pillars with proof
- tagline options
- CTA language
- short boilerplate
- medium boilerplate
- words and phrases to use
- words and phrases to avoid

For each major option, include a short rationale tied to strategy, audience, or
proof.

## Step 6: Review Gate

Before treating messaging as complete, confirm:

- Is the category understandable?
- Is the main value specific?
- Does each claim have proof or an assumption label?
- Does the language match the voice guide?
- Are taglines and CTAs usable rather than decorative?
- Are jargon and buzzwords removed unless the audience expects them?

## Step 7: Handoff

End with the next useful step:

- use a copywriting workflow for full pages, emails, ads, or social posts
- use `workflows/brand-visual-guidelines.md` if messaging needs visual-system
  implications
- use `workflows/brand-kit-generation.md` to package the messaging in a full
  brand kit
