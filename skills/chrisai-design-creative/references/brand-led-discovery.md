# Brand-Led Discovery

Use this reference when the skill needs to turn logos, style guides,
reference sites, and product assets into a concrete creative direction.

For products with an existing app or site design, use
[existing-design-sources](existing-design-sources.md) first. Brand-led
discovery is primary for greenfield work and secondary for existing-design
extension.

## Brand Inputs

Start with the logo or brand mark. Ask for a brand style guide when available.

Useful inputs:

- logo or brand mark
- brand style guide
- design source of truth for existing app or site work
- existing product screenshots
- existing marketing pages
- existing app or website
- icon, illustration, photo, pattern, or font assets
- color tokens or design-system tokens
- 3-5 reference websites or apps

Treat a brand style guide as authoritative unless the user explicitly wants a
departure.

For existing-design extension, treat the design source of truth as
authoritative before external reference sites.

## Logo Analysis

Capture concrete signals:

- dominant colors, secondary colors, and neutrals
- contrast behavior on light and dark backgrounds
- flat, gradient, transparent, or image-heavy treatment
- sharp, rounded, geometric, organic, dense, or sparse shape language
- symmetry, balance, and proportions
- symbol meaning and industry cues
- personality signals such as playful, premium, technical, practical,
  editorial, luxury, or operational
- motifs that can become UI details, backgrounds, dividers, icon treatments,
  or motion ideas

Do not overread the logo. Mark uncertain interpretations as tentative.

## Brand Guide Analysis

Extract constraints:

- approved colors
- typography rules
- spacing or layout rules
- logo usage rules
- image or illustration style
- voice and tone
- accessibility or contrast requirements
- forbidden treatments

When the style guide conflicts with a reference site, the style guide wins
unless the user says otherwise.

## Reference Analysis

For each reference site or app, identify what the user likes or dislikes before
borrowing from it.

Analyze:

- layout structure
- navigation model
- first-screen composition
- information density
- typography behavior
- color behavior
- button and form style
- card, table, or panel treatment
- image, icon, and illustration usage
- motion or interaction style
- onboarding flow
- conversion or task flow
- what seems useful for this project
- what should not be copied

## Commonalities

Commonalities reveal familiar category patterns and reduce cognitive load.

Look for repeated:

- hero layouts
- navigation placement
- dashboard structures
- content hierarchy
- CTA placement and labels
- trust signals
- onboarding patterns
- pricing or comparison structures
- visual density
- product screenshot treatments
- tone or personality

Preserve common patterns when they help users understand the interface faster.

## Differences

Differences reveal meaningful design choices.

Track:

- top navigation versus sidebar navigation
- minimal versus expressive surfaces
- card-heavy versus table-heavy layouts
- screenshot-led versus copy-led first screens
- guided wizard versus direct manipulation
- playful versus enterprise tone
- onboarding-heavy versus expert-first flow

Use differences to ask focused choices when the tradeoff affects the user's
audience or product strategy.

## Differentiation

Differentiate after brand and reference evidence are understood.

Prefer differentiation from:

- logo shape language
- brand color behavior
- a signature layout motif
- a distinctive screenshot or demo treatment
- a simpler version of a common flow
- clearer hierarchy than references
- better onboarding than references

Avoid novelty that makes the interface harder to understand.
