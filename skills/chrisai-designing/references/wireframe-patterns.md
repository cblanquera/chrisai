# Wireframe Patterns

Use this reference when choosing low-fidelity structures.

## Homepage Or Landing Page

Common structures:

- hero, proof, value sections, feature details, CTA
- hero, product screenshot area, use cases, social proof, CTA
- orientation lead, recommended path, section index, supporting details

Choose based on whether the page is mainly persuasive, orienting, or
task-starting.

## Documentation Surface

Common structures:

- purpose, quick start, core concepts, reference paths
- search-first docs index with categories
- handbook-style section list with summary cards

Keep downstream reading surfaces calmer and denser than promotional pages.

## App Page Or Screen

Common structures:

- top summary, filters, data table, detail drawer
- sidebar navigation, toolbar, content panel
- stepper or wizard for guided setup
- split view for list/detail workflows
- dashboard overview with status, tasks, and recent activity

Choose based on repeat use, user expertise, and information density.

## Multi-Step Flow

Common structures:

- linear stepper for required order
- hub-and-spoke when sections can be completed independently
- review screen before commit
- progressive disclosure for advanced settings

Keep choices visible before irreversible actions.

## Full-Scope Checklist

Before handing off to creative design, confirm the wireframes cover the full
requested scope:

- entry page or starting screen
- all requested destination screens
- required modals, drawers, or focused states
- empty, loading, error, success, and disabled states when they change layout
- mobile and desktop variants when responsive behavior affects structure
- navigation paths between screens
- review or confirmation screen before irreversible actions when applicable

If the scope is intentionally partial, say that clearly and do not treat it as
approved for full creative design.

## Modals And Drawers

Use a modal for focused interruption.

Use a drawer for side inspection or edit-in-context behavior.

Do not hide primary page navigation inside modal-only flows unless the user
goal is intentionally narrow.
