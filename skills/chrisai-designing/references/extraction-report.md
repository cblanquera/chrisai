# Extraction Report

Use this structure when handing design rules to another design or
implementation skill.

## Required Sections

### Sources

List what was inspected:

- URLs
- local files
- screenshots
- design files
- project tokens or components

Also list sources that were unavailable or unreadable.

### Authority

State the source-of-truth priority used and any conflicts found.

### Visual Rules

Summarize:

- colors and state colors
- typography and scale
- spacing and layout rhythm
- surface treatment
- radius, shadows, borders, and dividers
- icon, image, and illustration behavior

### Interface Rules

Summarize:

- navigation patterns
- form patterns
- table patterns
- card and panel patterns
- modal or drawer patterns
- empty, loading, success, error, warning, and disabled states

### Responsive Rules

Summarize visible desktop, tablet, and mobile behavior when available.

### Extension Constraints

State the rules the new page, screen, flow, or component must obey.

### Unknowns

List missing states, inaccessible files, unreadable formats, or assumptions
that still need user confirmation.
